⠋Q+A

What teams usually ask before connecting systems.

How does XBert know when to use an integration?

XBert starts from the customer's request. If the answer needs live availability, a customer record, a booking flow, or a follow-up action, it uses the connected system instead of relying only on static knowledge.

Can XBert work with custom APIs?

Yes. Custom API work can be scoped with support when the workflow goes beyond the core native connections.

How do permissions work?

You define the systems, fields, actions, and handoff moments. XBert should only act inside the boundaries you approve.

Do we need to replace our current software?

No. The point is to let XBert work with the software your team already knows, then keep the record and handoff clean.

What happens when XBert cannot safely act?

It escalates. Your team gets the conversation, key facts, and recommended next step instead of starting from a voicemail or a blank ticket.
